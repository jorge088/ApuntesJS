Funciones

Una función es un conjunto de intrucciones que realiza una tarea especifica y puede ser utilizado las veces que quieramos

Las funciones nos permiten dividir nuestro codigo en pequeñas partes.

Para declararlas se usa la palabra reservada function seguido por el nombre de la función,unos parentesis y el codigo que ejecuta entre llaves.

function nombre() {

// Codigo que ejecuta la funcion

)

nombre(); // Ejecuta la función

Las funciones pueden recibir valores para realizar sus operaciones, estos valores les son enviados cuando las ejecutamos y reciben el nombre de parametros.

Cuando declaramos las funciones, dentro de los parentesis escribimos la cantidad de parametros que recibe. Luego cuando ejecutamos, escribimos los valores de los parametros dentro de los parentesis. Recordá enviar los valores de los parametros en el mismo orden que fueron declarados.

function calcAreaRectangulo( base, altura ) {

let area = base * altura;

console.log(`El area del rectangulo es ${area}`);

)

calcAreaRectangulo(4,2); // base==4 altura==2

De esta forma podemos armar funciones que cumplan distintos propositos, según los valores que le enviemos.

Tambien las funciones funciones pueden devolver un valor, para hacerlo utiliza la palabra reservada return. Utilizando el return podemos crear funciones que realicen operaciones especificas y usar el valor que devuelvan para realizar otras operaciones o mostrarlo.

Continuando con el ejemplo de los numeros pares, podemos hacer una función que devuelva si un numero es par o no.

function esPar( num ) {

let par = (num % 2) == 0 ; // true si num es par

return par;

)

console.log( esPar(4) ); // true

console.log( esPar(7) ); // false

console.log( esPar(24) ); // true
  • Recuerda nombrar a las funciones con nombres simples que describan la tarea que realizan, usando camelCase.
  • Recuerda definir las funciones en la cabecera del codigo.
  • Las funciones deben realizar la operación que dicen realizar, no deben hacer operaciones extras. Por ejemplo: una función calcDescuento, solo deberia calcular el descuento y devolver su valor, no mostrarlo o hacer algo extra. Al valor del descuento lo mostramos fuera de la función.
  • Una función puede tener varios return. Cuando se ejecuta alguno, la función finaliza devolviendo el valor de ese return. Se lo suele usar para devoler un valor según alguna condición que definamos.
  • Recuerda que las funciones nos sirven para segmentar nuestro codigo en partes y de esta forma hacerlo más sencillo.

Funciones Flecha

Existe un tipo de funciones que se llaman Funciones anonimas que se declaran sin nombre y se asignan a una variable, luego para usarla hacemos referencia a esa variable.

const esPar = function ( num ) { return (num%2)===0 }

const areaRectangulo = function ( base, altura ) { return base * altura }

console.log( esPar(4) ); //true

console.log( areaRectangulo(4,2) ); // 8

Las Funciones flechas o Arrow Functions, reciben el nombre de funciones anonimas de sintaxis simplificada. Se agregaron en la versión EM6 de js, y utilizan una => en lugar de la palabra function.

const esPar = ( num ) => { (num%2)===0 };

const areaRectangulo = ( base, altura ) => base*altura ;

console.log( esPar(4) ); //true

console.log( areaRectangulo(4,2) ); // 8

El uso de las arrow functions tiene muchas ventajas.

  • Si el cuerpo de la función tiene una sola linea, se puede omitir las llaves { } y automaticamente se hace un return de esa linea por lo que se puede omitir escribir return.
  • Si la función no tiene parametros se escribe así ( ) => { }.
  • Si la función tiene solamente un parametros se lo puede indicar solamente por su nombre sin parentesis. Para el primer ejemplo sería: num => { }
  • Las arrow functions hacen que el codigo sea mucho más legible y claro de escribir.
  • En frameworks de js, como React, las arrow function se las usa mucho muchoo, te recomiendo aprenderlas.

Scope

El scope o alcance puede definirse como el alcance que tiene una variable dentro de tu codigo. El scope decide a qué variables tienes acceso en cada parte del codigo. Existén 2 tipos de scope, el local y el global.

Se habla de scope local cuando solo se puede acceder a un variable unicamente en cierta parte del codigo (dentro de una función o un bloque de codigo { } ). Estas variables se llaman variables locales. Si tratas de acceder a una variable local fuera de su ambito, tendrás un error que dice que la variable no está definida.

Se dice que una variable está en el scope global cuando está definida fuera de una función o bloque de codigo. Estas variables son llamadas variables globales y puedes acceder a ellas desde cualquier parte del codigo.
