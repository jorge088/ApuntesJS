Objetos

Un objeto es una colección de propiedades. Estas propiedades son una asociación entre un nombre (o clave) y un valor (cualquier tipo de dato). El valor de una propiedad puede ser una función, en este caso la propiedad se conoce como metodo. Se puede decir que los objetos están compuestos por un listado de pares clave-valor.

Los objetos en js, y en varios lenguajes de programación, son el equivalente a objetos en la vida real: un objeto tiene propiedades y funcionalidades.

En javascript, los objetos están en casi todos los aspectos del lenguaje, practicamente todo es un objeto. Por lo tanto, debemos comprenderlos.

En muchos lenguajes de programación, para crear un objeto se utiliza la palabra reservada new. En js también funciona:

const objeto = new Object(); // Esto crea un objeto 'genérico' vacío

Sin embargo, siempre que se pueda, en js se prefiere utilizar lo que se conoce como los objetos literales, que es un metodo abreviado para crear objetos directamente sin usar la palabra new.

Para crear un objeto literal, se usan las llaves { }.

const objeto = { }; // Crea un objeto vacío

Dentro de las llaves, cada propiedad está separada por una , (la ultima propiedad puede no llevar). Vamos a crear un objeto llamado 'producto', que tiene como propiedades a 'nombre', 'categoria' y 'precio'.

const producto = {

nombre : "Camiseta selección Argentina" ,

categoria : "Selección Argentina" ,

precio: 14000

}
  • La estructura de las propiedades es: "clave:valor".
  • Tanto nombre de los objetos y de las propiedades son sencibles a mayusculas y minusculas.

Acceso a las propiedades

Para acceder a las propiedades de un objeto existen dos formas, a través de la notación con puntos o a través de la notación con corchetes.

// Notación con puntos

console.log( productos.nombre ); // Camiseta selección Argentina

console.log( productos.categoria ); // Selección Argentina

// Notación con corchetes

console.log( productos["nombre"] ); // Camiseta selección Argentina

console.log( productos["precio"] ); // 14000

Modificar las propiedades

Para establecer o cambiar los valores de las propiedades se utiliza las mismas notaciones mencionadas.

// Notación con puntos

productos.nombre = "Camiseta Boca Juniors" ;

productos.categoria = "Futbol" ;

// Notación con corchetes

productos["nombre"] = "Camiseta Boca Juniors"

productos["precio"] = "Futbol"
  • Ambas notaciones funcionan de igual manera y puedes usar la que más te guste. La más utilizada en js es la notación con puntos.

Te dejo un ejemplo de cuando podria usarse la notación con corchetes:

const producto = {

nombre : "Campera de lluvia Adidas" ,

categoria : "Camperas" ,

precio: 18000

}

let llave = prompt("¿Que te gustaria saber del producto?")

// acceso por medio de la clave usando corchetes

alert( producto[ llave ] );

si ingresa "nombre", se muestra "Campera de lluvia Adidas"

si ingresa "precio", se muestra 18000
  • En el ejemplo anterior, usando la notación con puntos no nos habria funcionado.

Metodos

Se mencionó que las propiedades que tienen una función asignada se conocen como metodos, entonces, un metodo es una propiedad de un objeto que es una función. Los metodos son basicamente iguales a una función, con la diferencia de que solo pueden ser ejecutados dentro del mismo objeto.

const telefono = {

nombre : "Galaxy J6" ,

marca : "Samsung" ,

encendido : true,

apagar : function ( ) { this.encendido = false } , // Función anonima, utiliza this para actualizar el valor de la propiedad encendido

encender : function ( ) { this.encendido = true } ,

estaEncendido : function ( ) { console.log(this.encendido) } , // muestra el valor de la propiedad encendido

}

telefono.estaEncendido() ; //Ejecuta la funcion estaEncendido, muestra true

telefono.apagar()//ejecuto el metodo apagar

telefono.estaEncendido() ; //false

Función Constructor

Como alternativa, podes usar la función constructor para crear objetos. Para hacerlo debes definir el tipo de objeto usando una función constructora, luego para crear un objeto de ese tipo se usa el operador new (se crea una instancia de ese objeto).

Para definir un tipo de objeto, tenes que crear una función para el objeto que especifique su nombre(primera letra con mayuscula), propiedades y metodos. Ejemplo voy a definir un objeto llamado 'Producto':

// Función contructora

function Producto(nombre, categoria, precio) {

this.nombre = nombre;

this.categoria = categoria;

this.precio = precio;

this.mostrarDetalle = ( ) => console.log( `Nombre: ${ this.nombre }, Categoria: ${ this.categoria }, Precio: ${ this.precio }` ) ; // metodo, muestra en consola un texto con la información del objeto

}


// Creo 3 objetos del tipo Producto,envio como parametros de la función los valores de las propiedades del objeto.

let producto1 = new Producto( "Camiseta Selección Argentina", "Seleccion Argentina", 14000 );

let producto2 = new Producto( "Camiseta Boca Juniors", "Futbol", 12000 );

let producto3 = new Producto( "Camiseta River Plate", "Futbol", 12000 );

producto2.mostrarDetalle( ); // muestra "Nombre: Camiseta Boca Juniors, Categoria: Futbol, Precio: $12000"

La palabra this se usa para asignar los valores a las propiedades del objeto. La palabra this está enlazada con el nuevo objeto instanciado. Para nuestro ejemplo this se enlaza con los nuevos objetos 'producto1','producto2' y 'producto3', de esta forma les asigna sus valores correspondientes.

Clases

Las clases fueron introducidas en la versión de ES6, y lo que hacen es proveer una sintaxis mucho más clara y sencilla para crear objetos.

Para definir una clase, se realiza una declaración de clase. Para declarar una clase se utiliza la palabra reservada class y seguido, un nombre para la clase. Dentro de las llaves, se definen los metodos y el constructor de la clase.

class Producto {

constructor(nombre, categoria, precio) {

this.nombre = nombre;

this.categoria = categoria;

this.precio = precio;

}


mostrarDetalle( ) {

console.log( `Nombre: ${ this.nombre }, Categoria: ${ this.categoria }, Precio: ${ this.precio }` ) ;

}

}

La función constructora que conociamos, es reemplazada por el metodo contructor que cumple el mismo proposito de la función constructora. Recibe por parametros los valores que le vamos a asignar a las propiedades del objeto.

Todos los metodos que tenga el objeto se declaran sin utilizar la palabra function.

Cuando quieramos crear un objeto usando una clase, se utiliza la palabra new.

let producto1 = new Producto( "Camiseta Seleccion Argentina", "Seleccion Argentina", 14000 ) ;
  • Las clases nos ofrecen una forma más sencilla y ordenada para crear nuestros objetos.
  • Recuerda que el nombre de las clases inicia con mayusculas(Producto, Animal, Alumno, etc.).
  • Puedes pensar que son como una plantilla que contienen las propiedades y caracteristicas de un objeto.

Ejercicio 1

Definir un objeto literal de nombre 'deportista', que tiene las siguientes propiedades nombre, energia (inicia en 50), experiencia (inicia en 0).

Queremos pedirle al deportista que entrene, para eso tiene que tener un metodo de nombre 'entrenarHoras'. El metodo entrenarHoras recibe por parametro la cantidad de horas que va a entrenar. Luego le resta la energia al deportista la cantidad horas X5, e incrementa la experiencia la cantidad de horas X2. Agregar un control si al actualizar la energia del deportista es menor o igual a cero, mostrar un alerta con el mensaje 'El deportista está cansado para entrenar tanto! Elige menos horas' y no realiza acciones.

También queremos que el deportista descanse y recargue su energia. Para eso tiene que tener un metodo llamado 'descansar', que lo que hace es actualiza la energia del deportista a 50(valor maximo).

Luego de definir el objeto literal, probar las funciones y comprobar que los valores se actualicen correctamente.
